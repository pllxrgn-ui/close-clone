# Security review — 2026-07-16 (task 5h)

Read-only whole-codebase review (Explore agent). Result: **no critical vulnerabilities**; strong control posture. 2 medium + 3 low, all bounded.

## Findings

1. **MEDIUM — Blind SSRF, outbound webhook delivery.** `services/webhooks/service.ts` `validateUrl` only checks http/https; doesn't block loopback/private/link-local (169.254.169.254 metadata) or require https. Admin-gated + blind (delivery reads only status). Fix: reject private/loopback/link-local hosts, require https, pin resolved IP (anti-rebind). → FIXING.
2. **MEDIUM — Weak default SESSION_SECRET not fail-closed.** `config.ts` defaults to `dev-insecure-session-secret`, reused for cookie HMAC + AES OAuth-token key + unsubscribe HMAC. Unset in prod → admin-cookie forgery + token decryption. Fix: throw in production when absent/default. → FIXING.
3. **LOW — actorId from request body on POST /emails/send.** Documented pre-auth seam (`templates/access.ts`); real only if the deploy root fails to overwrite actorId with the authenticated principal. → deploy/WIRING.md note (derive actorId from request.user).
4. **LOW/hardening — global auth wiring deferred** (server.ts has no global requireSession; import RBAC optional). This is the D-031 deploy-composition wiring. → already tracked in deploy/WIRING.md.
5. **LOW/info — OIDC azp not checked for multi-audience tokens.** `oidc/id-token.ts audienceMatches`. Negligible for single-client Google Workspace. Fix: require azp===audience when aud is an array. → FIXING.
